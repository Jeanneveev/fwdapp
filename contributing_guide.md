# Contributing Guide (Fork → Clone → Edit → Pull Request)

This repository contains civic technology ideas for the Maryland Forward Party community. Contributions are welcome from anyone interested in building tools that improve civic participation, transparency, and organization.

Follow the steps below to contribute.

---

## 1. Fork the Repository

Start by creating your own copy of the repository:

- Go to the repository on GitHub
- Click the **“Fork”** button (top right)
- This creates a copy under your GitHub account

---

## 2. Clone Your Fork Locally

Clone your fork to your local machine so you can make changes:

```bash
git clone https://github.com/YOUR_USERNAME/REPO_NAME.git
cd REPO_NAME
```

(Optional but recommended) Add the original repo as an upstream remote:

```bash
git remote add upstream https://github.com/ORIGINAL_OWNER/REPO_NAME.git
```

---

## 3. Create a Branch for Your Work

Always create a new branch before making changes:

```bash
git checkout -b feature/your-feature-name
```

Examples:
- `feature/petition-app-prototype`
- `feature/bill-visualization-ui`
- `fix/spreadsheet-protection-script`

---

## 4. Make Your Changes

You can contribute in several ways based on the ideas in this repo:

### Example Contribution Areas

- Build or prototype one of the listed projects:
  - Prioritized Problems Mobile App
  - Monthly Dues Ballot System
  - Petition Signature Prefill Tool
  - Voting Spreadsheet Protection Tool
  - Platform Topic Graph Database
  - Maryland Bills Transparency Tool
  - Civic Data Visualizations

- Improve documentation
- Add new project ideas
- Link prototypes or demos
- Propose architecture or technologies

Be sure your changes are:
- Clear and readable
- Well-organized
- Relevant to civic tech or the project goals

---

## 5. Commit Your Changes

Stage and commit your work:

```bash
git add .
git commit -m "Add: short description of your change"
```

---

## 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

---

## 7. Open a Pull Request

- Go to your fork on GitHub
- Click **“Compare & pull request”**
- Submit the PR to the original repository

### In your PR description:
- Explain what you built or changed
- Reference the relevant idea/project
- Include screenshots or links if applicable
- Mention any design decisions or tradeoffs

---

## 8. Participate in Review

- Respond to feedback from maintainers
- Make updates if requested
- Keep discussion focused and constructive

---

## Alternative: Start with an Issue

If you’re not ready to code yet:

1. Open an issue describing:
   - The idea you want to work on
   - Proposed approach or tech stack
2. Get feedback from others
3. Then proceed with implementation

---
